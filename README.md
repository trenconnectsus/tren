# TREN Website

Tennessee Research & Education Network — static site built with Hugo, hosted on GitHub Pages.

## Setup

1. Clone `git@github.com:trenconnectsus/tren.git`
2. Copy these files into the repo root
3. Enable GitHub Pages in repo Settings → Pages → Source: GitHub Actions
4. Push to `master` — the site deploys automatically

## Where to change things

| What | File |
|---|---|
| Site URL | `hugo.yaml` → `baseURL` |
| Nav links | `hugo.yaml` → `params.nav` |
| Social links | `hugo.yaml` → `params.social` (icons: linkedin, facebook, x, youtube) |
| Institutions | `hugo.yaml` → `params.institutions` |
| Homepage stats | `data/stats.yaml` |
| Hero image | `static/images/hero.jpg` (replace this file) |
| NSF award number | `hugo.yaml` → `params.nsf_award` |
| Hugo version | `.github/workflows/deploy.yml` → `HUGO_VERSION` |

## Adding content

**News item:**
```
content/news/your-item-name/
  index.md
  photo.jpg        ← optional
```

**Event (one-off):**
```
content/events/your-event-name/
  index.md
```

**Page:**
```
content/about/governance/
  index.md
```

## Recurring events

Instead of creating a new content file for every occurrence (e.g. the bi-weekly
call), one content file (`content/events/biweekly-community-call/index.md`)
generates all future occurrences automatically. Front matter fields:

| Field | Meaning |
|---|---|
| `date` | Anchor date — the first occurrence |
| `recurrence_days` | Repeat every N days |
| `recurrence_count` | How many occurrences to generate forward from the anchor (bump this or move `date` forward once you run out) |
| `upcoming_count` | How many future occurrences to display |
| `cancelled_dates` | List of `"YYYY-MM-DD"` strings — that occurrence stays listed but marked "Cancelled" instead of disappearing |

To cancel a call: add its date to `cancelled_dates` and deploy. To add a new
recurring series, copy this file's front matter pattern into a new event page.

## Team page

`content/team/*/index.md` supports:
- `pi: true` — shown in the "Principal Investigator" section
- `status: "current"` / `status: "alumni"` + `category: "phd" / "ms" / "undergraduate"` — grouped accordingly

## Local preview

```bash
brew install hugo     # macOS
make serve
```

Open http://localhost:1313

## Deployment

```bash
make deploy
```

Builds locally (catches errors early), stages + commits all changes with an
autogenerated message, and pushes to `master`. That push (or the nightly
scheduled rebuild — see `.github/workflows/deploy.yml`) triggers GitHub
Actions to build and deploy to GitHub Pages. Check the Actions tab for build
status.
